# Writer Sub-Reference — 看点密集度纪律（Density Discipline）

## 功能说明

本文件覆盖 Writer 系统 prompt 中**唯一一段按"3 档字数硬尺"算密度**的段：

- **§7.5 看点密集度（番茄老师鎏旗，硬尺）**——300 字 / 500 字 / 1000-1500 字 三档密度，决定本章是否被 Auditor 判"水文 / 节奏断"。

启用条件：恒启（所有章号都吃这条硬尺）。在 SKILL Writer prompt 拼装顺序里位于 §7 黄金开场纪律之后、§8 题材规则 + 主角铁律之前。本段移植自 inkos `writer-prompts.ts` 的"看点密集度（番茄老师鎏旗，硬尺）"段（commit `b1cc3a7`）。

## 与上下游段的关系

§7.5 与 §7 黄金开场纪律是叠加（不冲突）：黄金开场决定"开场 300 字 + 第 1 章前 800 字"该长什么样；§7.5 决定**全章范围**该有多少看点 / 钩子 / 完整悬念。前 3 章两段一起注；第 4 章起 §7 整段移除，§7.5 仍然恒启。

§7.5 与 §4 写作工艺卡里"段落字数 40-120 字"的硬尺**互补**：

- §4 管"段内"——一段叙事多长才算合规
- §7.5 管"段外"——把章拆成 300 / 500 / 1000-1500 字的窗口，每个窗口必须命中一种密度货

两段都不达标 = Auditor critical。

## §7.5 verbatim prompt（原样注入 Writer 系统 prompt）

```
## 看点密集度（番茄老师鎏旗，硬尺）

本章正文从头到尾必须满足以下节奏，写完后自检：

- **每 300 字至少 1 个爽点**：小看点、有趣的梗、炸裂的小情节、反套路小动作、暧昧台词、情绪拉扯都算
- **每 500 字至少 1 个钩子**：引发读者"接下来怎样"的小悬念；不要求揭开，要求抛出
- **每 1000-1500 字至少 1 个完整悬念**：一组"问题—蓄力—未解"的结构，给读者追下去的理由
- 不靠密度堆砌糊弄——单章里的爽点 / 钩子 / 悬念必须服务于本章 goal，不能是和主线无关的孤立段落
- 如果某段连续 300 字以上是环境、回忆、议论、心理独白而没有推进主线或制造看点，就是水文，必须删或改
```

## 自检清单（写完初稿前过一遍）

- [ ] 把全章按 300 字切片，每片至少有 1 个爽点 / 反套路小动作 / 暧昧台词 / 情绪拉扯
- [ ] 把全章按 500 字切片，每片至少抛出 1 个钩子（不要求揭开）
- [ ] 全章至少有 1 组完整"问题 — 蓄力 — 未解"结构
- [ ] 没有任何一段连续 300 字以上是纯环境 / 回忆 / 议论 / 心理独白且不推主线
- [ ] 爽点 / 钩子 / 悬念都和本章 goal 有因果链，不是孤立段落

任何一条不通过 → 改稿，不要进 Settler。

## 下游联动

| 下游 | 怎么用 §7.5 |
|---|---|
| Auditor dim 18（节奏紧张度） | 三档密度未达标的章被扣分；连续 2 章未达标触发 [long-span-fatigue.md](../long-span-fatigue.md) §"密度疲劳"补救动作 |
| Reviser | 收到 Auditor "密度不足"反馈时，按 §7.5 三档窗口逐个补 |
| Settler | 不直接消费；密度问题不进真理文件 |

## 注意事项

- **密度服务于主线**：单章塞 5 个爽点和 0 个推进主线 ≠ 高密度，是堆砌。爽点必须是本章 goal 的副产品。
- **环境描写不是反密度**：≤ 300 字的氛围 / 心理 / 议论段不犯规；只有连续超过 300 字、没夹任何看点 / 钩子，才算水文。
- **§7.5 与 §4 段落字数硬尺互补**：§4 让段不要碎、§7.5 让章不要稀。两条都通过才算密度合规。
